# instaCmaker 캐러셀 JSON 생성기 (지침 세트)

instaCmaker 스튜디오의 **JSON 불러오기**에 바로 붙여넣을
**요소(레이어) JSON + 캡션 + 해시태그 + DM 문구**를 생성하는 Claude 역할(Project) 정의 문서 세트입니다.

주제 하나만 주면 → `element-schema.md` 포맷에 맞는 **요소 JSON**이 나오고 → 스튜디오의
**JSON 불러오기**에 붙여넣으면 캐러셀이 한 번에 구성됩니다(사진·캐릭터는 그 뒤 직접 조정).

> ⚠️ 구버전(21종 template id) 포맷은 폐기되었습니다. 이제 `template/fields/arrays`가 아니라
> **`background + elements`** 요소 포맷입니다. (옛 `template-catalog.md` → 새 `element-schema.md`)

## 파일 구성

| 파일 | 용도 |
|------|------|
| `SYSTEM_PROMPT.md` | Claude Project **커스텀 인스트럭션**에 붙여넣는 역할 정의 |
| `element-schema.md` | **스키마 정본** — 요소 타입·좌표계·폰트/팔레트·**레이아웃 프리셋 6종 JSON**. 프로젝트 지식으로 첨부 |
| `copywriting-skill.md` | **전략·톤·엉클비 디자인 시스템** 플레이북. 프로젝트 지식으로 첨부 |
| `README.md` | (이 문서) 개요·사용법 |

## 사용법 (Claude.ai Project)

1. Claude.ai에서 **새 Project** 생성.
2. Project **커스텀 인스트럭션**에 `SYSTEM_PROMPT.md` 본문을 붙여넣는다.
3. **프로젝트 지식(파일)** 으로 `element-schema.md` 와 `copywriting-skill.md` 를 첨부.
4. 대화창에 **`주제: OOO`** 형식으로 입력 (선택: `타깃/톤/장수/유형`).
   - 예) `주제: AI로 블로그 글 빨리 쓰기 / 타깃: 1인 창작자 / 6장 / 정보형`
5. 출력에서 **① JSON 코드블록을 복사**.
6. 스튜디오(`/insta`) → 상단 **JSON 불러오기** → 텍스트 영역에 붙여넣고 **불러오기**.
   - "현재 구성을 덮어씁니다" 확인 → 캐러셀이 한 번에 구성됨.
7. 슬라이드별로 문구·색을 다듬고, 필요한 사진/캐릭터를 조정한 뒤 **이 장 PNG / 전체 ZIP** 다운로드.
8. ② 캡션 / ③ 해시태그 / ④ DM 문구를 게시·자동응답에 사용.

> 팁: JSON이 안 불러와지면 거의 **따옴표/콤마**거나 **숫자를 문자열로** 쓴 경우다. 셀프체크(`JSON.parse 가능?`, `좌표가 숫자?`)를 다시 돌려달라고 하면 된다. 링크는 슬라이드가 아니라 DM(④)에만.

## 입력 모드

- **기본 생성**: 주제/스크립트 → 요소 JSON.
- **(C) URL 분석**: 유튜브/블로그/기사 URL → 원문 전체 분석 → 요소 JSON.
- **(B) 이미지 분해**: 그리드 배경 시트 1장 → 슬라이드별 배경 파일 N장(에디터에서 배경 이미지로 업로드).

## 향후 자동화 (개념)

주제 풀(시트/노션)에서 매일 주제 1개 → 이 역할로 생성 → 메일 발송. **검수 후 게시**가 원칙(사실/컴플라이언스/이미지 확인). 자동화도 슬라이드 URL 미노출 셀프체크 유지.
