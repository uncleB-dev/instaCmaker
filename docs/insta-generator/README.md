# EYEshare 인스타 캐러셀 JSON 생성기

EYEshare(`eyeshare.unclebstudio.com`)의 `/insta` 캐러셀 스튜디오에 바로 붙여넣을
**캐러셀 JSON + 캡션 + 해시태그 + DM 문구**를 생성하는 전용 Claude 역할(세션/Project) 정의 문서 세트입니다.

주제 하나만 주면 → 스키마에 100% 맞는 `/insta` 구성 JSON이 나오고 → `/insta`의
**📥 구성 불러오기**에 붙여넣으면 6장 캐러셀이 한 번에 구성됩니다(이미지는 그 뒤 직접 업로드).

## 파일 구성

| 파일 | 용도 |
|------|------|
| `SYSTEM_PROMPT.md` | Claude Project **커스텀 인스트럭션**에 붙여넣는 역할 정의 |
| `template-catalog.md` | **스키마 정본** — 21개 템플릿의 유효 template id·필드·배열·개수 + 골드 예시. 프로젝트 지식으로 첨부 |
| `copywriting-skill.md` | **전략·톤·EYEshare 팩트** 플레이북. 프로젝트 지식으로 첨부 |
| `README.md` | (이 문서) 개요·사용법·향후 자동화 |

## 사용법 (Claude.ai Project)

1. Claude.ai에서 **새 Project** 생성.
2. Project의 **커스텀 인스트럭션**에 `SYSTEM_PROMPT.md` 본문을 붙여넣는다.
3. **프로젝트 지식(파일)** 으로 `template-catalog.md` 와 `copywriting-skill.md` 두 파일을 첨부한다.
4. 대화창에 **`주제: OOO`** 형식으로 입력 (선택: `타깃/톤/장수/판매여부`도 함께).
   - 예) `주제: 전화 상담의 한계 / 타깃: 보험설계사 / 6장 / 정보형`
5. 나온 출력에서 **① JSON 코드블록을 복사**한다.
6. `eyeshare.unclebstudio.com/insta` → **📥 구성 불러오기 / 내보내기** 펼치기 → 텍스트 영역에 붙여넣고 **불러오기**.
   - "현재 구성을 덮어씁니다" 확인 → 캐러셀이 한 번에 구성됨.
7. 슬라이드별로 **배경 사진/삽입 이미지**를 올리고(`showcase`·`mockup`은 이미지 자리), 문구를 다듬은 뒤 **전체 PNG(ZIP)** 다운로드.
8. ② 캡션 / ③ 해시태그 / ④ DM 문구를 인스타 게시·자동응답에 사용.

> 팁: JSON이 안 불러와지면 거의 따옴표/콤마 문제다. 출력의 셀프체크 항목(`JSON.parse 가능?`)을 다시 돌려달라고 하면 된다. 또한 슬라이드에는 링크를 넣지 않는 게 원칙이므로(퍼널: 댓글 → DM), 링크는 DM 문구(④)에만 둔다.

## (향후) Claude Cowork 자동화 아웃라인

매일 아침 캐러셀 초안을 자동 생성해 받아보는 파이프라인. **개념 수준**의 설계만 적습니다.

1. **주제 자동 선정** — 주제 풀(스프레드시트/노션) 또는 콘텐츠 캘린더에서 그날 주제 1개를 뽑는다.
2. **이 역할로 생성** — 위 SYSTEM_PROMPT 역할에 주제를 넣어 `JSON + 캡션 + 해시태그 + DM`을 생성.
3. **이메일 발송** — 결과를 담당자(예: `mybohumben@gmail.com`)에게 매일 아침 메일로 전송.

필요한 것:
- **주제 소스**: 주제 풀/캘린더(시트·노션 등)와 "오늘의 주제" 선택 규칙.
- **이메일 연동**: 발송 채널(Gmail 등)·수신자.
- **스케줄**: 매일 아침 정해진 시각 트리거.

주의:
- **검수 후 게시**가 원칙. 자동 생성물은 초안 — 사람이 사실/컴플라이언스(과장·규제 표현)·이미지 자리를 확인하고 발행.
- 자동화가 슬라이드에 URL을 넣지 않도록(퍼널 원칙) 셀프체크를 유지.
