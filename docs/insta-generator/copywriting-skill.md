# copywriting-skill.md — 인스타 캐러셀 카피라이팅 플레이북 (범용)

> 한국어 정보형·세일즈형 인스타 캐러셀을 위한 재사용 전략집.
> 스키마는 `template-catalog.md`, **전략·톤·문구**는 이 문서가 담당.
> ⚠️ **콘텐츠는 독립적**이 기본. 특정 브랜드(예: EYEshare)는 **"연결" 요청이 있을 때만** §8 부록을 참고해 가볍게 엮는다.

---

## 1. 콘텐츠 원칙

- **주제가 주인공**: 그 자체로 유익한, 저장·공유할 만한 콘텐츠를 만든다.
- **독립성**: 기본적으로 어떤 브랜드도 홍보하지 않는다. 노하우·사실 중심.
- **연결은 옵션**: 사용자가 "연결: ○○○"라고 할 때만, 흐름을 해치지 않는 선에서 **마지막(또는 한 장)**에 자연스럽게. 억지 연결·본문 광고화 금지.
- **사실 기반**: 지어내지 않는다. 수치·효과는 검증된 것만.

## 2. 표준 6장 플로우 (기본 골격)

| # | 역할 | 추천 템플릿 |
|---|------|------------------|
| 1 | **훅 커버** | `cover` / `statcover` |
| 2 | 문제·페인 | `problem` / `compare` |
| 3 | 개념·정의 (필요 시) | `define` / `impact` |
| 4 | 방법·기능 | `features` / `howto` / `steps` |
| 5 | 혜택·이유·근거 | `card3` / `checklist` / `stats` |
| 6 | **CTA** | `cta` |

- **정보형**: 1 훅 → 2~5 개념/방법/근거 → 6 CTA. (길면 `toc`로 목차 한 장)
- **세일즈형**: 끝을 `qna` + `notice`(유의사항)로 신뢰 쌓고 CTA(7~8장).
- 보통 **6장**. 가벼우면 5장, 설득이 길면 7~8장.

## 3. 훅(1장) 공식 — 5택 1

1. **숫자·권위**: "○○의 80%가 놓치는…", "2배" → `statcover`.
2. **결과 약속**: "끝나면 ○○하게 됩니다" → `cover`.
3. **무료·신규**: "무료로 받아가세요", "드디어" → `cover`.
4. **질문·페인**: "아직도 ○○하세요?" → `cover`/`problem`.
5. **캐릭터·시점**: "3년차 ○○인데…".

> 본문 습관: **칩(태그) + 굵은 질문형/단정형 헤드라인 + 짧은 설명 + 다음 장 유도**.
> 프롬프트·스크립트 공유는 `promptcard`(monospace + `[변수]`). 변수는 대괄호 `[주제]` `[자료]` 식.

## 4. CTA 퍼널 (계정 성장 — 브랜드 중립)

- 공식: **"팔로우 + 댓글[키워드] → DM으로 자료/링크"** + Save/Share 유도 + 핸들 노출.
- 키워드는 짧고 주제 연관되게: 예) `'자료'`, `'템플릿'`, `'체험'`.
- **슬라이드에 실제 URL·링크를 직접 노출하지 않는다.** 링크는 **DM으로만**. `cta`의 `button`/`sub`에 주소를 쓰지 말 것 — "댓글 → DM" 표현만.
- DM으로 주는 "자료/링크"는 **주제에 맞는 무료 자료**가 기본. 브랜드 연결 시에만 그 브랜드 링크.
- 캡션 끝에도 동일 CTA를 한 번 더.

## 5. 톤 · 금지

- **무압박·도움 먼저.** 설득보다 "이래서 도움이 된다"를 보여주기. 존댓말.
- **과장·허위 금지.** 검증 안 된 수치·효과 단정 금지.
- **"무제한"·"무조건" 류 금지.**
- **규제 주제 주의(보험/금융/의료/법률 등)**: 최고·최저·1등·"보장"·단정·공포 마케팅 금지. 심의 표현 주의, **사실 기반**.
- 이모지는 거들기용으로 절제(슬라이드당 과다 금지). 카드 `emoji`는 1개.

## 6. 헤드라인 / 카피 길이 가이드

| 위치 | 길이 |
|------|------|
| 커버 헤드라인 (`headline`) | **2~3줄**, 짧게. `\n`으로 끊어 리듬 만들기 |
| 본문 설명 (`desc`·`definition` 등) | **2~3문장** 이내 |
| 카드 `desc` / 행 `desc` | **1줄** (스마트폰에서 한눈에) |
| 칩(`chip`) | 4~12자 태그 |
| `problem.question` | 굵은 질문 2~3줄 |
| `cta.button` | 행동 한 줄 (예: "팔로우 + 댓글 '자료' → DM") |

- 한국어 **존댓말** 통일. 구어체 OK("…하나요?", "…돼요").
- 줄바꿈으로 의미 단위를 끊어 가독성↑ (`multiline` 필드).

## 7. 주제 → 슬라이드 매핑 (시퀀스 예시 — 브랜드 무관)

### A. 교육형 — "○○ 잘하는 법"
`cover` → `problem` → `define` → `features` → `stats` → `cta`

### B. 방법 안내형 — "○○ 이렇게 하세요"
`cover` → `features` → `howto` → `showcase` → `qna` → `cta`

### C. 설득형 — "왜 ○○해야 하는가"
`statcover` → `problem` → `compare` → `card3` → `notice` → `cta`

> 어떤 주제든: **1장 훅 / 중간 문제·방법·근거 / 마지막 CTA** 구조를 유지하고, 카탈로그의 템플릿 중 주제 성격에 맞는 것을 고르면 됩니다.

## 8. 부록 — 브랜드 연결 (요청 시에만 사용)

> ⚠️ 아래는 **"연결: EYEshare"가 지정됐을 때만** 참고. **기본 콘텐츠에는 넣지 않는다.** 다른 브랜드는 사용자가 준 사실만 사용한다.

### EYEshare 팩트 (이 사실만, 지어내지 말 것)
- **한 줄**: "통화 중에도 같은 화면을 1:1 실시간 공유"하는 SaaS.
- **운영사**: 엉클비스튜디오(대표 남윤영). 핸들 `@unclebstudio`. 도메인 `eyeshare.unclebstudio.com`.
- **가격**: 무료 베타.
- **핵심 기능**: 앱 설치 없이 링크/QR/6자리 코드 입장(카메라·마이크 미사용, 화면만) · 실시간 어노테이션(펜·형광펜·포인터로 짚어가며) · 상담 녹음(불완전판매 예방·설명의무 증빙, 호스트 기기에만 저장) · 끝나면 표시 포함 자료 일괄 전달 · ~80ms 동기화 · 분할보기 · PC 호스트 + 모바일 고객.
- **타깃**: 보험설계사 등 전화 상담·영업 직군(확장: 과외·상담·교육).

### 연결 방식
- 본문(2~5장)은 **주제 자체**에 충실하게 쓰고, **마지막 CTA 또는 한 장**에서만 "이런 게 필요하면 ○○" 식으로 가볍게 연결.
- 슬라이드 본문을 브랜드 광고로 채우지 말 것. DM(④)에서 링크/체험을 안내.
- 규제 주제에서 EYEshare는 "상담을 돕는 도구"이지 보험상품·수익을 약속하지 않음 — 단정·보장 표현 금지.

### 엉클비 프롬프트 도구함 — 핵심 가치 (도구 홍보 연결 시)

> "연결: 엉클비 프롬프트 도구함(`/make` 랜딩·프로필)"이 지정됐을 때만. 도구(랜딩페이지/프로필 사진 프롬프트 생성기)를 홍보하는 캐러셀·릴스에서 페인과 훅을 잡는 기준.

**핵심 메시지(이대로 외워 슬로건처럼 반복하진 말 것 — 페인→해결 흐름으로 풀어 쓰기)**
- 남이 만든 프롬프트는 결국 '남이 원한 것' → **내가 원하는 결과가 한 번에 안 나온다.**
- AI는 한 번 나온 결과물에 **수정을 반복할수록 점점 망가진다** → 그래서 **'처음 프롬프트'의 정확도**가 전부.
- 이 도구는 용도·스타일·디테일을 **내가 직접 골라 담은 맞춤 프롬프트**를 만들어 줌 → **처음 한 번에** 원하는 결과에 가까워진다. = **자유도 + 정확도 = 원샷(한 번에).**

**페인을 어디에 잡을지(중요)**: 도구 홍보 시 페인을 "사진이 없다 / 소개 페이지가 없다"가 아니라 **"원하는 걸 한 번에 못 뽑는다"**로 잡는다. ("페이지·사진 없음"은 부차적 — 진짜 통증은 *복붙해도 내 맘 같지 않다*.)

**추천 페인/훅 문구 (그대로/변형해서 — `cover`·`problem` 헤드라인이나 캡션 훅용)**
- "프롬프트 복붙해도 원하는 게 **한 번에** 안 나오죠 → **남이 만든** 프롬프트라 그래요."
- "AI는 **수정 반복하면 망가져요** → 그래서 **처음**이 정확해야 합니다."
- "남이 원한 결과 말고, **내가 원한 결과** — 골라서 담는 맞춤 프롬프트."
- "용도·스타일·디테일 **내가 고르면** → 처음 한 번에 가까워집니다."

**키워드**: 맞춤 / 한 번에(원샷) / 자유도 + 정확도 / 처음 프롬프트 / 내가 고른.

- 슬라이드에 도구 URL을 직접 노출하지 않는다(§4) — 링크·체험은 DM으로만. 규제 주제 톤(보장·단정 금지)·"무제한" 류 금지는 그대로 적용.
