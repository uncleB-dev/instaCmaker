# copywriting-skill.md — 인스타 캐러셀 카피·디자인 플레이북 (뭉이)

> 한국어 정보형·세일즈형 인스타 캐러셀을 위한 재사용 전략집.
> 스키마·좌표는 `element-schema.md`, **전략·톤·문구·디자인 톤**은 이 문서가 담당.
> ⚠️ **콘텐츠는 독립적**이 기본. 특정 브랜드는 **"연결" 요청이 있을 때만** §8을 참고해 가볍게 엮는다.

---

## 1. 콘텐츠 원칙

- **주제가 주인공**: 그 자체로 저장·공유할 만한 유익한 콘텐츠.
- **독립성**: 기본적으로 어떤 브랜드도 홍보하지 않는다. 노하우·사실 중심.
- **연결은 옵션**: "연결: ○○○" 지정 시에만, 흐름을 해치지 않는 선에서 마지막(또는 한 장)에 자연스럽게.
- **사실 기반**: 지어내지 않는다. 수치·효과는 검증된 것만.

## 2. 표준 6장 플로우 (프리셋 매핑)

> 프리셋 = `element-schema.md` §8 의 (A)~(F).

| # | 역할 | 추천 프리셋 |
|---|------|------------|
| 1 | **훅 커버** | (A) 커버 |
| 2 | 문제·페인 / 개념 | (B) 본문+캐릭터 · (D) 비교 |
| 3 | 방법·기능 | (B) 본문 · (E) 리스트 |
| 4 | 근거·구성 | (E) 리스트 · (D) 비교 |
| 5 | 실전·복붙 | (C) 프롬프트 카드 |
| 6 | **CTA** | (F) 엔딩 CTA |

- **정보형**: 1 훅 → 2~5 개념/방법/근거 → 6 CTA.
- **세일즈형**: 끝에 비교·리스트로 신뢰 쌓고 CTA(7~8장).
- 보통 **6장**. 가벼우면 4~5장, 설득이 길면 7~8장. (페이지 인디케이터·워터마크 번호를 장수에 맞춤)

## 3. 훅(1장) 공식 — 5택 1

1. **숫자·권위**: "○○의 80%가 놓치는…", "2배".
2. **결과 약속**: "끝나면 ○○하게 됩니다".
3. **무료·신규**: "무료로 받아가세요", "드디어".
4. **질문·페인**: "아직도 ○○하세요?".
5. **캐릭터·시점**: "3년차 ○○인데…". (뭉이 캐릭터와 잘 맞음)

> 커버 헤드라인은 **2~3줄**, 강조어는 코랄로 한 줄 분리. 키커(영문 대문자, 코랄, 자간 4)로 카테고리를 얹는다.

## 4. CTA 퍼널 (계정 성장 — 브랜드 중립)

- 공식: **"팔로우 + 댓글[키워드] → DM으로 자료/링크"** + Save/Share 유도 + 핸들 노출.
- 키워드는 짧고 주제 연관: 예) `'자료'`, `'템플릿'`, `'GPT'`.
- **슬라이드에 실제 URL·링크 노출 금지.** 링크는 **DM(④)에서만.**
- 엔딩 CTA(F)의 다크 박스에는 **핸들**을, 링크는 넣지 않는다.
- 캡션 끝에도 동일 CTA를 한 번 더.

## 5. 톤 · 금지

- **무압박·도움 먼저.** 존댓말. 구어체 OK("…하나요?", "…돼요").
- **과장·허위 금지.** 검증 안 된 수치·효과 단정 금지.
- **"무제한"·"무조건" 류 금지.**
- **규제 주제(보험/금융/의료/법률 등)**: 최고·1등·"보장"·단정·공포 마케팅 금지. 사실 기반.
- 이모지·장식(✦)은 절제. 강조는 색(코랄)·굵기·형광펜으로.

## 6. 카피 길이 가이드

| 위치 | 길이 |
|------|------|
| 키커 | 영문/한글 짧게(4~16자), 대문자+자간 |
| 커버 헤드라인 | 2~3줄, `\n`으로 리듬 |
| 본문 설명 | 2~3문장 이내, `\n`으로 의미 단위 |
| 카드/행 설명 | 1~2줄 |
| 리스트 항목 | 한 줄 |
| 엔딩 박스 | 핸들 한 줄 |

- 줄바꿈으로 끊어 가독성↑. 한국어 존댓말 통일.

## 7. 뭉이 디자인 시스템 (요소 작성 기준)

- **배경**: 크림 `#EDE6D4` 기본. 강조/마무리 슬라이드는 다크 `#15151A`(글자 밝게).
- **컬러**: 헤드라인·본문 네이비 `#1B1B22`, 강조어·키커·포인트 코랄 `#E8553A`, 서브·푸터 머드 `#8A8577`, 형광펜 옐로 `#F4C84A`.
- **타이포 역할**:
  - 키커: `size` 26~30, `weight` 800, 코랄, `letterSpacing` 4 (영문 대문자 권장).
  - 헤드라인: `size` 96~130, `weight` 900, `lineHeight` 1.05~1.1. 강조 줄은 코랄로 **분리**.
  - 본문: `size` 32~40, `weight` 500, 머드/네이비, `lineHeight` 1.4~1.5.
  - 영문 강조 라벨엔 `fontFamily:"display"`(이탤릭 세리프), 코드/프롬프트엔 `"mono"`.
- **반복 요소**: 우상단 페이지 인디케이터(`01 — 06`), 배경 거대 워터마크 숫자(`#E4DCC9`, 맨 뒤 레이어), 좌하단 핸들 + 우하단 스튜디오 푸터, 다음 유도 "swipe to read →"(코랄 이탤릭).
- **컴포넌트 레시피**:
  - 비교 카드 = `roundRect`(라이트 `#E3DBC8` / 다크 `#15151A`) + 제목 + 본문.
  - 프롬프트 박스 = `roundRect`(`#15151A`, radius 28) + "PROMPT" 라벨(코랄) + mono 본문(`#E7E3D8`).
  - 엔딩 핸들 박스 = `roundRect`(`#15151A`, radius 75) + 핸들(흰색, 가운데).
  - 형광펜 강조 = 텍스트에 `highlight:"#F4C84A"`.

## 8. 주제 → 프리셋 시퀀스 (예시)

### A. 교육형 — "○○ 잘하는 법"
(A) 커버 → (B) 본문(개념) → (E) 리스트(방법) → (D) 비교(전·후) → (C) 프롬프트 → (F) CTA

### B. 방법 안내형 — "○○ 이렇게 하세요"
(A) 커버 → (B) 본문(왜) → (E) 리스트(스텝) → (C) 프롬프트 → (F) CTA

### C. 설득형 — "왜 ○○해야 하는가"
(A) 커버 → (D) 비교 → (B) 본문(근거) → (E) 리스트 → (F) CTA

> 어떤 주제든 **1장 훅 / 중간 문제·방법·근거 / 마지막 CTA** 구조 유지. 프리셋을 골라 문구만 채운다.

---

## 9. 부록 — 브랜드 연결 (요청 시에만)

> ⚠️ "연결: ○○○"가 지정됐을 때만. **기본 콘텐츠에는 넣지 않는다.**

- 본문(중간 장)은 **주제 자체**에 충실, **마지막 CTA 또는 한 장**에서만 가볍게 연결.
- 슬라이드 본문을 브랜드 광고로 채우지 말 것. 링크·체험은 DM(④)에서.
- 브랜드 팩트는 **사용자가 제공한 사실만** 사용한다. 부족하면 **사용자에게 묻고**, 추측·창작 금지.
- 규제 주제에서 브랜드는 "돕는 도구"이지 성과·수익을 약속하지 않음 — 단정·보장 표현 금지.
